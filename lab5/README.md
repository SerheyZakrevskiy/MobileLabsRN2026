# Лабораторна робота №5

## Тема

Побудова навігації у React Native із використанням бібліотеки Expo Router.

## Мета роботи

Ознайомитися з концепцією file-based маршрутизації в мобільних застосунках та навчитися будувати публічні, захищені й динамічні маршрути за допомогою Expo Router.

## Назва застосунку

**Каталог товарів з авторизацією**

## Використані технології

- React Native
- Expo
- Expo Router
- TypeScript
- React Context
- FlatList
- Dynamic routes

## Реалізований функціонал

У застосунку реалізовано:

- file-based маршрутизацію через директорію `app`;
- публічні маршрути для входу та реєстрації;
- глобальний контекст авторизації `AuthContext`;
- функції `login`, `register` та `logout`;
- захищену групу маршрутів `app/(app)`;
- автоматичне перенаправлення неавторизованого користувача на екран входу;
- головний екран із каталогом товарів;
- список товарів через `FlatList`;
- локальні зображення товарів;
- перехід до сторінки деталей товару через `<Link asChild>`;
- динамічний маршрут `details/[id]`;
- отримання параметра `id` через `useLocalSearchParams`;
- екран обробки неіснуючих маршрутів `+not-found`;
- кнопку виходу з акаунту.

## Опис основних файлів

### `app/_layout.tsx`

Кореневий layout застосунку. У ньому підключається `AuthProvider`, який робить стан авторизації доступним для всіх екранів застосунку.

### `context/AuthContext.tsx`

Файл глобального контексту авторизації. Містить:

- `isAuthenticated` — стан авторизації користувача;
- `login()` — функцію входу;
- `register()` — функцію реєстрації;
- `logout()` — функцію виходу з акаунту;
- `useAuth()` — хук для доступу до контексту.

### `app/(auth)/login.tsx`

Публічний екран входу. Містить поля Email і Пароль, кнопку **Увійти** та посилання на сторінку реєстрації.

### `app/(auth)/register.tsx`

Публічний екран реєстрації. Містить поля Імʼя, Email, Пароль, Підтвердження паролю, кнопку **Зареєструватися** та посилання для повернення на сторінку входу.

### `app/(app)/_layout.tsx`

Layout захищеної частини застосунку. Якщо користувач не авторизований, виконується перенаправлення на екран входу.

### `app/(app)/index.tsx`

Головний екран після авторизації. На ньому відображається каталог товарів. Кожна картка містить зображення, назву, ціну та короткий опис товару. Також є кнопка **Вийти**.

### `app/(app)/details/[id].tsx`

Екран деталей товару. Це динамічний маршрут, який отримує `id` товару через `useLocalSearchParams`, знаходить відповідний товар у масиві даних і відображає повну інформацію.

### `app/+not-found.tsx`

Екран помилки для неіснуючих маршрутів. Відображає повідомлення **Екран не знайдено** та кнопку повернення на головну сторінку.

### `data/products.ts`

Файл із тестовими даними товарів. Кожен товар має:

- `id`;
- `name`;
- `price`;
- `image`;
- `description`.

## Встановлення та запуск

### 1. Перейти в папку лабораторної роботи

```bash
cd lab5
```

### 2. Встановити залежності

```bash
npm install
```

### 3. Запустити застосунок

```bash
npx expo start
```

або з очищенням кешу:

```bash
npx expo start -c
```

### 4. Відкрити застосунок

Застосунок можна відкрити:

- через Expo Go на телефоні;
- через Android Emulator;
- через iOS Simulator.

## Інструкція користування

1. Запустити застосунок.
2. На екрані входу ввести Email і Пароль.
3. Натиснути кнопку **Увійти**.
4. Після входу відкриється каталог товарів.
5. Натиснути на картку товару.
6. Відкриється сторінка деталей товару.
7. Натиснути кнопку повернення назад, щоб повернутися до каталогу.
8. Натиснути **Вийти**, щоб завершити сесію.
9. Після виходу користувач повертається на екран входу.
10. На екрані реєстрації можна створити новий акаунт і перейти до каталогу.

## Контрольні запитання

### 1. Яким чином за допомогою Expo Router реалізується перенаправлення неавторизованого користувача?

Перенаправлення реалізується у файлі `_layout.tsx` захищеної групи маршрутів. Якщо значення `isAuthenticated` дорівнює `false`, повертається компонент `<Redirect href="/(auth)/login" />`. У такому випадку користувач не може потрапити до захищених екранів і автоматично переходить на сторінку входу.

### 2. У чому полягає різниця між використанням компонента `<Link>` та метода `router.push()`?

`<Link>` використовується як декларативний спосіб навігації. Його зручно застосовувати в інтерфейсі, наприклад для кнопок, посилань або карток товарів.

`router.push()` використовується як імперативний спосіб навігації. Його зручно викликати всередині функцій, наприклад після авторизації, реєстрації або натискання на кнопку.

### 3. Як працюють динамічні маршрути в Expo Router і як отримати передані параметри?

Динамічні маршрути створюються за допомогою квадратних дужок у назві файлу. Наприклад, файл `[id].tsx` відповідає маршрутам типу `/details/1`, `/details/2` тощо.

Переданий параметр можна отримати за допомогою хука `useLocalSearchParams()`. У лабораторній роботі через цей хук отримується `id` товару, після чого відповідний товар знаходиться у масиві даних.

### 4. Чому стан авторизації доцільно зберігати у глобальному контексті, а не в локальному стані компонента?

Стан авторизації потрібен одразу в багатьох частинах застосунку: на екранах входу, реєстрації, у захищеному layout та при виході з акаунту. Якщо зберігати цей стан локально в одному компоненті, інші компоненти не матимуть до нього зручного доступу. React Context дозволяє зробити стан авторизації глобальним і використовувати його в будь-якому екрані.

### 5. Для чого використовуються групи маршрутів `(folderName)` і як вони впливають на URL-адресу?

Групи маршрутів використовуються для логічного розділення екранів проєкту. Наприклад, у лабораторній роботі є група `(auth)` для публічних екранів і група `(app)` для захищених екранів.

Назва групи в круглих дужках не додається до кінцевої адреси маршруту. Тобто вона впливає лише на структуру проєкту, але не змінює URL-адресу для користувача.

## Очікуваний результат

У результаті виконання лабораторної роботи створено мобільний застосунок із file-based маршрутизацією на базі Expo Router. Застосунок має публічні екрани входу та реєстрації, захищений каталог товарів, динамічні сторінки деталей товарів і сторінку обробки неіснуючих маршрутів.

## Висновок

Під час виконання лабораторної роботи було розроблено мобільний застосунок на React Native з використанням Expo Router. Було досліджено принцип file-based маршрутизації, використання layout-файлів, груп маршрутів, захищених маршрутів і динамічних параметрів. Також було реалізовано глобальний контекст авторизації, який дозволяє керувати доступом користувача до захищеної частини застосунку.
